# 目标检测评估指标与训练曲线解读

## 文档用途

本文用于回答模型训练页面中的 Box Loss、Cls Loss、DFL Loss、Precision、Recall、mAP、PR 曲线、混淆矩阵和 Epoch 等问题。指标必须结合数据集、阈值和类别分布解释，不能只看一个数值下结论。

## 从预测框到 TP、FP、FN

评估目标检测时，通常先在同一个类别内比较预测框和人工标注框：

1. 当预测类别正确，且预测框与标注框的 IoU 达到评价阈值时，记为 TP（真正例）。
2. 没有匹配到任何标注框的预测框，记为 FP（假正例，也称误检）。
3. 没有被任何预测框匹配到的标注框，记为 FN（假负例，也称漏检）。

目标检测中背景区域数量极大，因此 TN（真负例）通常不像普通二分类那样作为核心指标使用。

## Precision（精确率）

```text
Precision = TP / (TP + FP)
```

Precision 关注“模型报出的结果中有多少是对的”。Precision 较低通常说明误检较多，可能与阈值过低、类别相似、背景干扰或训练数据不足有关。

## Recall（召回率）

```text
Recall = TP / (TP + FN)
```

Recall 关注“真实存在的目标中找回了多少”。Recall 较低通常说明漏检较多，可能与阈值过高、小目标、遮挡、图片模糊或训练样本覆盖不足有关。

## Precision 与 Recall 的权衡

降低置信度阈值通常会保留更多预测框：Recall 可能提高，但 FP 增多会使 Precision 降低。提高阈值则相反。

对于病害初筛，较好的实践是保留适中的阈值，并让低置信度结果进入人工复核；不应只追求单一指标最大化。

## F1 分数

```text
F1 = 2 × Precision × Recall / (Precision + Recall)
```

F1 是 Precision 和 Recall 的调和平均数。当两者都较高且相对平衡时，F1 较高。F1 曲线常用于帮助选择一个折中置信度阈值，但最终阈值仍应按业务风险决定。

## AP 与 mAP

AP（Average Precision）是一个类别在不同置信度阈值下的 Precision-Recall 曲线面积。mAP（mean Average Precision）是多个类别 AP 的平均值。

- `mAP@50`：以 IoU=0.50 判断预测框是否匹配标注框。
- `mAP@50-95`：在 0.50、0.55、0.60 一直到 0.95 的多个 IoU 阈值上计算并平均，要求定位更准确，数值通常低于 mAP@50。

例：如果模型能判断出“这是番茄早疫病”但框的位置不够准确，mAP@50 可能不错，而 mAP@50-95 会更低。

## 每类别指标为什么重要

总体 mAP 是平均值，可能掩盖少数类别表现不佳的问题。应同时查看每个类别的 AP：

- 高发类别通常样本多，指标可能较稳定。
- 少样本类别可能出现 AP 波动大、漏检多或误检多。
- 外观相近的类别，例如不同番茄叶斑类病害，需要检查混淆矩阵和具体错误样本。

本项目训练页面的类别名称应优先使用中文显示名，内部英文类别名仍用于保持模型与数据集的一致性。

## 混淆矩阵

混淆矩阵的行和列分别表示真实类别与预测类别。对角线数值越高，说明该类别被正确识别的样本越多；非对角线较高的位置表示两个类别容易混淆。

阅读混淆矩阵时应重点关注：

1. 某个病害是否经常被预测成另一个相似病害。
2. 健康叶片是否被误判为病害，或病叶是否被误判为健康。
3. 是否存在某一类别样本太少导致整行或整列不稳定。
4. 是否需要补充真实场景、不同光照和不同生育期的困难样本。

## 训练损失

训练页面可能包含以下损失项：

- Box Loss：边界框位置和大小的误差。下降通常说明定位在改善。
- Cls Loss：类别预测误差。下降通常说明类别区分在改善。
- DFL Loss：Distribution Focal Loss 相关的定位质量损失。下降通常说明边界框回归趋于稳定。

损失值的绝对大小与模型、数据、损失定义有关，不能跨项目简单比较。更有价值的是观察趋势以及验证集指标是否同步改善。

## 如何判断过拟合和欠拟合

### 可能欠拟合

- 训练损失和验证指标都较差。
- 训练多轮后仍没有明显改善。
- 可能原因：模型容量不足、训练轮数不足、图片尺寸太小、标签质量差或学习率设置不合适。

### 可能过拟合

- 训练损失继续下降，但验证集 Precision、Recall 或 mAP 停滞甚至下降。
- 模型在训练集表现很好，在新的田间图片上表现较差。
- 可能原因：数据量少、类别分布单一、训练轮数过多或增强不足。

应使用独立验证集和测试集，而不是只看训练曲线。

## Epoch、进度与结果文件

一个 Epoch 表示模型完整遍历一次训练集。Ultralytics 的 `results.csv` 中 Epoch 已是从 1 开始的轮数，导入平台时不能再次加 1。

本项目训练页面会将任务进度和曲线限制在配置的 `epochs` 范围内。例如任务配置为 80 轮，界面和指标列表应显示 1 到 80，不应出现 `81/80`。

## 评估前的检查清单

- 训练集、验证集、测试集之间没有重复或近似重复图片。
- 类别 ID 与 `data.yaml` 中类别顺序一致。
- 标签框没有越界、空框或错误类别。
- 各类别样本量足以覆盖典型外观和真实拍摄条件。
- 评估使用的阈值、图片尺寸和数据划分被记录下来。
- 同时保存模型权重、`results.csv`、混淆矩阵、PR 曲线和验证报告。
