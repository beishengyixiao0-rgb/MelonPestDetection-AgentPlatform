# 项目开发记录
本文档用于记录每个成员的提交内容，方便团队成员协作和后续维护。
每次提交到仓库时需要在本文件里追加自己的提交内容，注意不要删除文件里的已有内容。

## 成员提交模板

- `path/to/file`

**修改文件**

- `path/to/file`

**删除文件**
- `path/to/file`

1. 是否需要安装后端依赖(`backend/requirements.txt` 是否发生变化)：是/否
```powershell
cd backend
.\.venv\Scripts\activate
pip install -r requirements.txt
```
1. 是否需要安装前端依赖(`frontend/package.json` 或 `frontend/package-lock.json` 是否发生变化)：是/否
```powershell
cd frontend
npm install
```
1. 是否需要执行数据库迁移(`backend/alembic/versions/*.py` 是否发生变化)：是/否
```powershell
cd backend
.\.venv\Scripts\activate
alembic upgrade head
```
1. 是否需要重启 Docker(`docker-compose.yml`是否发生变化)：是/否
```powershell
docker compose up -d
```
1. 是否需要修改 `.env`：是/否
  - 需要新增/修改的配置： `KEY=value`
  - 敏感信息（API Key、密码、Token等）不要上传
2. 是否新增或修改了API：是/否
#### API 记录规范
后端成员新增或修改 API 时，必须记录以下内容：
- 接口名称
- 请求方法和路径
- 是否需要登录鉴权
- 请求参数
- 响应数据
- 错误状态码
- 前端调用建议
示例：
````md
### API：方法名称（如获取当前用户信息）
- 方法：GET
- 路径：/api/auth/me
- 鉴权：需要 `Authorization: Bearer <token>`/不需要
- 请求参数：无
- 后端文件：`backend/app/api/xxx.py`
- 前端封装：`frontend/src/api/xxx.js` 的 `xxxApi()`
- 调用页面/组件：`frontend/src/views/XxxPage.vue`
- 成功响应：
```json
{
  "id": 1,
  "username": "testuser",
  "email": "test@example.com",
  "roles": ["user"]
}
```
- 错误响应
```json
{
  "code": 400,
  "message": "错误信息",
  "data": null
}

```
- 前端调用示例
```js
import { xxxApi } from '@/api/xxx'
const res = await xxxApi(params)
```
