# YOLO 目标检测基础知识

## 文档用途与边界

本文用于回答本项目中的 YOLO、检测框、置信度、IoU、NMS、训练和视频跟踪等技术问题。它解释的是通用目标检测原理及本项目的使用方式，不代表某张图片已经被确诊为某种病害。

本项目使用 Ultralytics YOLO11 系列模型检测植物叶片病害类别。模型只能在训练类别范围内进行识别；对于训练集未覆盖的病害、营养缺素、药害、机械伤或多种胁迫并存的情形，结果需要人工复核。

## YOLO 是什么

YOLO（You Only Look Once）是一类单阶段目标检测模型。它将一张图片输入网络，在一次前向推理中同时输出候选目标的位置、类别和置信度，因此适合需要较快响应的图片、视频和摄像头场景。

传统两阶段检测器通常先生成候选区域，再对候选区域分类和回归；YOLO 直接从整张图像预测目标。单阶段模型通常推理更快，但在小目标、遮挡、类别相似或训练数据不足时仍可能产生误检和漏检。

## 一次检测经历了什么

```text
原始图片
  -> 尺寸缩放、色彩和格式转换
  -> YOLO 网络提取多尺度特征
  -> 预测候选框、类别分数和位置
  -> 置信度过滤
  -> NMS 去除同图重复框
  -> 返回类别、置信度、bbox 和标注图
```

常见组成如下：

- Backbone：从图片中提取纹理、边缘、形状等特征。
- Neck：融合不同尺度的特征，使模型同时看到局部细节和整体结构。
- Detection Head：预测类别、目标位置和置信度。
- Post-processing：根据 `conf` 和 `iou` 阈值筛选结果并执行 NMS。

用户通常不需要直接操作网络结构；实际可调参数主要是图片尺寸、置信度阈值、IoU 阈值，以及训练时的轮数、批大小和数据质量。

## 检测框和坐标

检测框通常以像素坐标 `[x1, y1, x2, y2]` 返回：

- `(x1, y1)`：左上角坐标。
- `(x2, y2)`：右下角坐标。
- 原点在图片左上角，向右为 x 增大，向下为 y 增大。
- 框宽度为 `x2 - x1`，高度为 `y2 - y1`。

YOLO 标签文件常使用归一化格式：`class_id center_x center_y width height`，其中坐标和尺寸通常除以图片宽高，范围接近 0 到 1。平台返回给前端的检测结果已转换为更直观的像素框。

## 置信度（Confidence）

置信度表示模型对一个候选检测框的相对确信程度，通常在 0 到 1 之间。它不是实验室诊断概率，也不能单独证明病害存在。

### 如何理解阈值

系统使用 `conf` 阈值过滤低置信度框：

- 较高阈值，例如 0.60：保留更少、更保守的结果；通常误检减少，但漏检可能增加。
- 中等阈值，例如 0.25 到 0.50：适合日常初筛，但应结合原图复核。
- 较低阈值，例如 0.10：保留更多候选框，适合人工复查，不适合直接做自动结论。

阈值选择必须结合业务目的。若目的是尽量不漏掉疑似病叶，可适当降低阈值并增加人工复核；若目的是减少无关提示，可提高阈值。

## IoU（Intersection over Union，交并比）

IoU 用于衡量两个边界框的重叠程度：

```text
IoU = 两个框的交集面积 / 两个框的并集面积
```

- IoU = 1：两个框完全重合。
- IoU = 0：两个框没有交集。
- IoU 越大，两个框的位置和大小越接近。

IoU 在本项目中有三类用途：

1. 评估：判断预测框是否与人工标注框匹配。
2. NMS：删除同一图片中高度重叠的重复框。
3. 视频去重降级：当跟踪器没有返回轨迹 ID 时，比较相邻采样帧中同类别框的重叠度。

IoU 阈值不是越高越好。阈值过低可能误把相邻叶片或病斑合并；阈值过高则可能保留同一目标的多个重叠框。

## NMS（非极大值抑制）

同一目标可能产生多个候选框。NMS 的典型过程是：

1. 按置信度从高到低排列候选框。
2. 保留当前置信度最高的框。
3. 计算其余框与该框的 IoU。
4. 删除 IoU 高于设定阈值的重复候选框。
5. 重复上述步骤直到没有候选框。

NMS 仅解决同一张图片的重复框。视频中同一目标在不同帧重复出现，不能靠 NMS 解决，需要使用跟踪 ID 或跨帧匹配。

## 类别、显示名与模型边界

模型内部类别名通常是英文，例如 `Tomato leaf late blight`。平台会返回：

- `class_name`：模型内部英文类别名。
- `class_name_cn`：中文映射名。
- `class_name_display`：按当前显示语言给用户看的类别名。
- `class_counts`：内部类别统计。
- `class_counts_display`：界面显示语言对应的统计。

默认显示语言为中文。模型类别名和中文显示名的映射在 `DetectionConfig` 中维护，避免为了切换界面语言而改变模型类别 ID。

## 图片质量与数据分布

模型效果不只由网络版本决定，图片与训练数据的匹配程度通常更重要。以下情况会降低可靠性：

- 图片模糊、过曝、严重阴影、压缩失真或分辨率过低。
- 叶片被遮挡、只拍到很小局部，或病斑不清晰。
- 真实场景的背景、光照、品种、生育期与训练数据差异很大。
- 同一叶片同时存在多种病害、虫害或非病原性伤害。
- 训练集类别不平衡，少数类别样本不足。

提高可靠性的办法包括：拍摄清晰的正面和背面叶片、包含病斑边缘和整叶、收集本地真实场景数据、持续补充人工复核后的训练样本。

## 训练中的常见参数

- `epochs`：完整遍历训练数据的次数。轮数增加不一定持续提升效果，过多可能过拟合。
- `batch_size`：每次参数更新使用的图片数量。受显存或内存限制。
- `img_size`：输入图片边长。更大尺寸可能保留更多细节，但计算更慢。
- `lr0`：初始学习率。过大可能不稳定，过小可能收敛缓慢。
- `optimizer`：优化器，用于根据损失更新模型参数。
- 数据增强：如翻转、缩放、色彩扰动和拼接。增强应贴近真实拍摄条件，不能破坏病斑特征。

训练曲线应结合验证集指标观察。仅训练损失下降并不等于模型在新图片上表现更好。

## 视频检测与 ByteTrack

视频检测通常不对每一帧推理，而是按一定间隔采样。为避免同一个叶片或病害目标在多个采样帧被反复计数，系统使用 ByteTrack 生成 `track_id`：

- 首次看到一个新的轨迹 ID 时，计入视频唯一目标总数。
- 后续帧出现相同 ID 时，只更新位置和标注，不重复计数。
- 单帧可见的目标数量与整段视频唯一目标总数不同。

跟踪适合连续画面。镜头切换、严重遮挡、快速移动或采样间隔过大，可能导致轨迹中断并产生新 ID。

## 常见误解

| 误解 | 正确理解 |
|---|---|
| 置信度 0.90 就是 90% 确诊 | 置信度是模型内部评分，仍要结合样本质量和现场信息。 |
| 没有检测框就一定没有病害 | 可能是漏检、拍摄不清、阈值过高或类别不在训练集内。 |
| NMS 可以解决视频重复计数 | NMS 只处理同一帧重叠框，视频重复需要跟踪。 |
| 训练轮数越多越好 | 轮数过多可能过拟合，应观察验证集曲线。 |
| mAP 高就适合所有田间场景 | 评估集和真实田间环境差异可能造成性能下降。 |
